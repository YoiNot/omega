# Benchmark baselines (2026-07-09)

Measured on a 4-core laptop (Node 20, single thread). These are baselines for regression
detection, not optimization targets. Higher hz = faster (more iterations/sec).

## @omega/engine-core

| Benchmark                       | hz        | mean (ms) |
|---------------------------------|-----------|-----------|
| Rng.nextF64 x 1000              | ~1,628    | 0.614     |
| Rng.nextInt(0,100) x 1000       | ~1,678    | 0.596     |
| ECS spawn 1000 (2 components)   | ~4,637    | 0.216     |
| ECS query 1000                  | ~3,503    | 0.180     |

## @omega/engine-math

| Benchmark                       | hz        | mean (ms) |
|---------------------------------|-----------|-----------|
| Mat4.multiply x 1000            | ~7,156    | 0.140     |

## Notes
- All generators/sim run on the Rng path above; ~1.6M f64 draws/sec sets the ceiling for
  per-frame seeded work.
- 4.6k entity spawns/sec indicates the ECS can host large worlds before allocation becomes
  the bottleneck (future: job system + memory pools per ADR/roadmap).
- Re-run with `npm run bench` and update this table on perf-relevant changes.
